# B6x 系列芯片 Flash 及 SRAM 说明

## 1. Flash 空间大小

Flash 共有三种配置，具体型号对应大小如下：

| 型号 | Flash 大小 |
| :--- | :--- |
| B61 / B62 | 128KB |
| B63 / B66 | 256KB |
| B68 | 512KB |

> **注意**：Flash 内部空间配置详见 `@B6x-Flash-Map.xlsx`。

---

## 2. SRAM 空间配置

所有型号具有相同的 SRAM 配置，总计32KB。SRAM 分为三块空间，地址映射如下：

| 名称 | 起始地址 | 结束地址 |
| :--- | :--- | :--- |
| **SRAM1** | `0x20003000` | `0x20007FFF` |
| **SRAM2** | `0x20008000` | `0x20009FFF` |
| **SRAM3** | `0x2000A000` | `0x2000AFFF` |

> **⚠️ 重要提示**：
> **SRAM3** 通常作为 **Cache** 使用，可配置为普通 SRAM。若配置为普通 SRAM，会影响代码在 Flash 里的运行速度。

---

## 3. SRAM 分配模式

SRAM 有以下几种配置方式，具体分配如下：

### 3.1 模式 1：不配置 BLE 协议栈
*   **SRAM1 和 SRAM2**：作为用户空间，为用户使用。
*   **SRAM3**：作为 Cache 使用。

### 3.2 模式 2：配置 BLE 协议栈（主 + 从共 3 个连接）
*   **BLE 库文件**：`ble6.lib`
*   **空间分配表**：

| SRAM 空间内容 | 长度（16 进制） |
| :--- | :--- |
| 中断向量表 | `0x98` |
| 用户空间 1 | `0x1D68` |
| 蓝牙协议堆 | `0x2C00` |
| 系统堆栈 | `0x600` |
| 蓝牙硬件与软件交换区 | `0x0FD8` |
| 蓝牙协议栈 | `0x304` |
| 用户空间 2 | `0xD24` |
| Cache | `0x1000` |

### 3.3 模式 3：配置 BLE 协议栈（主 或 从 1 个连接）
*   **BLE 库文件**：`ble6.lib`
*   **空间分配表**：

| SRAM 空间内容 | 长度（16 进制） |
| :--- | :--- |
| 中断向量表 | `0x98` |
| 用户空间 1 | `0x3169` |
| 蓝牙协议堆 | `0x1800` |
| 系统堆栈 | `0x600` |
| 蓝牙硬件与软件交换区 | `0x0FD8` |
| 蓝牙协议栈 | `0x304` |
| 用户空间 2 | `0xD24` |
| Cache | `0x1000` |

### 3.4 模式 4：配置 BLE 协议栈（仅从设备 1 个连接）
*   **BLE 库文件**：`ble6_lite.lib`
*   **空间分配表**：

| SRAM 空间内容 | 长度（16 进制） |
| :--- | :--- |
| 中断向量表 | `0x98` |
| 用户空间 1 | `0x4968` |
| 系统堆栈 | `0x600` |
| 蓝牙硬件与软件交换区 | `0x0BAC` |
| 蓝牙协议堆 | `0xC00` |
| 蓝牙协议栈 | `0x608` |
| 用户空间 2 | `0x24C` |
| Cache | `0x1000` |

### 3.5 模式 5：配置 BLE 协议栈（主 + 从共 6 个连接）
*   **BLE 库文件**：`ble6_8act_6con.lib`
*   **空间分配表**：

| SRAM 空间内容 | 长度（16 进制） |
| :--- | :--- |
| 中断向量表 | `0x98` |
| 用户空间 1 | `0x1D68` |
| 蓝牙协议堆 | `0x2C00` |
| 系统堆栈 | `0x600` |
| 蓝牙硬件与软件交换区 | `0x1590` |
| 蓝牙协议栈 | `0x3B4` |
| 用户空间 2 | `0x6BD` |
| Cache | `0x1000` |
