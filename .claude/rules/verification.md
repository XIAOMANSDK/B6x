# 完成前验证规则

> **铁律：没有新鲜的验证证据，就不能声称完成。**

## 红旗词 — 看到这些词时必须停下来运行验证

| 红旗词 | 要求动作 |
|--------|---------|
| "应该"、"可能"、"看起来" | 停下来，运行验证命令 |
| "Done!"、"完成！"、"修好了！" | 在说之前，是否有证据？ |
| "代码逻辑没问题" | 编译过了吗？ |
| "配置应该正确" | 运行 validate_config 了吗？ |
| "测试通过了" | 展示测试输出 |

## 验证矩阵速查

| 工作类型 | 最低验证要求 |
|---------|-------------|
| 驱动代码修改 | `/b6-build` 编译通过 (0 Error) |
| 驱动功能验证 | `/b6-auto-debug` RTT 输出符合预期 |
| BLE Profile 修改 | 编译通过 + 连接测试 |
| 硬件引脚配置 | `/b6-validate-hardware` 0 Error |
| 代码提交前 | `/b6-code-review` 无高优先级问题 |

## 规则

1. 本消息中没有运行过验证命令，就不能声称它通过了
2. 跳过任何验证步骤 = 未验证
3. 如果验证失败：陈述实际状态并附上证据，不做断言
4. 详细验证流程见 `/b6_verification-before-completion` skill
